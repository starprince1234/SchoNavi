# 计算机类竞赛规则知识库

> 最近核验：2026-06-30。计算机赛事经常按赛道独立发布规则，报名时应核对编程语言、运行环境、AI 工具、开源许可、数据许可和团队人数。

## 二、ICPC

- **名称纠错**：赛事现称 ICPC（International Collegiate Programming Contest）；“ACM-ICPC”是历史名称，旧资料可保留括注但不应当作当前正式名。
- **形式**：典型区域赛由同校 3 名学生组成一队，共用一台计算机，在限时内解决多道算法题；具体资格、区域、语言和罚时规则看当季 regional rules。
- **路径**：校内选拔、区域赛、区域冠军赛/洲际环节和 World Finals 的路径会调整，不写固定月份。
- **训练**：按专题训练后转向虚拟赛；记录首读、首交、错误罚时、卡题时间和队内交接。
- **合规**：比赛环境允许携带的纸质资料、IDE、模板和联网限制逐场核对。

## 三、CCPC

- 由中国大学生程序设计竞赛组委会组织，通常包括分站赛、女生专场、总决赛等安排，以当届通知为准。
- 题型和团队机试形式与 ICPC 接近，但资格、名额和晋级体系独立，不能把 CCPC 奖项写成 ICPC 奖项。
- 2025 赛季第十一届总决赛于 2026 年 4 月举行；后续赛季日期查看 [CCPC 官网](https://ccpc.io/)。

## 四、蓝桥杯

- 软件、电子和视觉艺术等赛项随届次变化；应按参赛组别核对院校/学历限制、比赛语言和环境。
- 省赛获奖与国赛晋级规则以当届章程为准，不再统一写“省一必进国赛”或固定奖项层级。
- 不承诺获奖带来企业招聘绿色通道。学校是否认定、如何加分必须查询校内文件。

## 五、中国大学生计算机设计大赛（4C）

2026 年第十九届由大赛组委会主办，面向当年在籍本科生，设置 11 个大类：

1. 软件应用与开发。
2. 微课与 AI 辅助教学。
3. 物联网应用与物联网创新转化（创业实践）。
4. 大数据应用。
5. 人工智能应用。
6. 信息可视化设计。
7. 数媒静态设计。
8. 数媒动漫与短片。
9. 数媒游戏与交互设计。
10. 计算机音乐创作。
11. 国际生“汉学”数智创作。

2026 起部分规则和分组有变化，且 AI+ 类别对可使用工具可能有专门清单。必须按具体大类规程准备，不能只看总通知。

[2026 大赛通知](https://jsjds.blcu.edu.cn/info/1042/2294.htm)

## 六、中国高校计算机大赛（C4）

C4 是系列赛，不是一个统一题型的单场比赛。2024 目录列出的子赛包括大数据挑战赛、团体程序设计天梯赛、移动应用创新赛、网络技术挑战赛、人工智能创意赛。各子赛的主办承办、资格、时间、平台和奖项独立。

- **大数据挑战赛**：重数据处理、验证、算法效果和复现。
- **团体程序设计天梯赛**：个人提交计入团队成绩，按当届组别与计分规则训练。
- **移动应用创新赛**：重产品完成度、交互、平台规范和现场展示。
- **网络技术挑战赛**：重网络设计、协议、系统实现和命题要求。
- **人工智能创意赛**：重真实问题、AI 方法、系统原型和伦理合规。

## 七、“中国软件杯”与服务外包大赛

### 中国软件杯

- 题目通常由企业提供，但每届命题企业、技术栈、知识产权和验收标准变化。
- 交付应包括需求分析、架构、代码、测试、部署、用户手册、演示和答辩。
- 不把“能跑 Demo”当完成：需要异常处理、权限、安全、性能和可维护性证据。

### 中国大学生服务外包创新创业大赛

- 可能包含企业命题和创业实践等类别；先确认题目是否要求指定平台、数据、接口或商业计划。
- 公开仓库前检查命题保密、第三方代码许可、数据许可和企业资料。

## 八、全国大学生计算机系统能力大赛

- 由全国高等学校计算机教育研究会、系统能力培养研究专家组和发起高校等组织，不能简单写成“教育部计算机教指委主办”。
- 包含操作系统、编译系统、处理器/体系结构、系统优化等不同赛道，入口和规则可能分散。
- 2026 赛事信息可从[系统能力大赛平台](https://pra.educg.net/)和[系统能力产学研网站](https://www.csc-he.cn/)核对。
- 验收重点：正确性、性能、资源、可移植性、测试覆盖、设计文档和可复现实验。

## 九、AI 与数据赛事

### 数据拆分

- 隐藏测试集存在时，不得通过人工探测、泄漏或提交反馈反推标签。
- 时间序列按时间切分；用户/设备数据按实体分组，避免同一实体跨训练和验证。
- 任何外部数据、预训练模型和人工标注都先确认规则是否允许。

### AI 工具

- 生成代码必须经过测试、许可证检查和人工理解。
- 不向外部模型上传未公开赛题、私有数据、账号、密钥或企业材料。
- 记录模型名称、版本、日期、用途和关键交互；按赛事要求披露。
- 禁止用多账号、人工代答、购买代码或共享赛中答案。

## 十、软件作品验收清单

- [ ] 从干净环境可按 README 部署。
- [ ] 依赖版本、数据下载和模型权重来源明确。
- [ ] 单元/集成/端到端测试覆盖关键路径。
- [ ] 无硬编码密钥、个人路径和演示专用假数据。
- [ ] 权限、输入校验、错误处理和日志满足基本安全要求。
- [ ] 指标与基线在相同数据和硬件条件下比较。
- [ ] 第三方代码、字体、图片、音频、模型和数据许可可用。
- [ ] 视频、PPT、论文和实际版本一致。
