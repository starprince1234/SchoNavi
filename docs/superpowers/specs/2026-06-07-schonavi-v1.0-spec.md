# SchoNavi V1.0 正式版本 Spec

- 版本：v1（2026-06-07）
- 依赖：主设计文档 + V0.1 + V0.2 spec
- 目标：**面向真实使用 + 接后端就绪**——补齐流式对话、登录与云同步抽象、导师对比、隐私合规、埋点与崩溃监控接入点，并打通「切到真实后端」的路径。

---

## 1. 范围

### 本阶段做（In Scope）

1. 流式对话（Mock 模拟逐字 + 中断 + 重试 + 完成态）。
2. 登录完善 + 云端同步抽象（`SyncService`，无后端时本地实现）。
3. 收藏/历史「本地 + 云」抽象（多设备同步占位）。
4. 导师对比（基础双人对比）。
5. 反馈闭环（提交态 + 可选历史反馈查看）。
6. 埋点（`AnalyticsService` + 事件接入 + Stub/可选 Firebase 占位）。
7. 崩溃监控接入点（Sentry/Crashlytics 预留）。
8. 隐私政策 / 用户协议 / 关于 / 数据删除入口。
9. 设置页完善。
10. 接真实后端准备（`Http*Repository` 骨架 + `dataSource=http` 走通）。

### 本阶段不做（开放问题约束）

简历解析 / 申请规划（Q7）、iOS 上架（Q9）、付费会员（Q11）、推送通知（Q12）。

---

## 2. 功能细化

### 2.1 流式对话（`features/chat`）

- `ChatRepository.streamReply(...) → Stream<ChatChunk>`；Mock 切片定时 emit 模拟逐字。
- UI：逐段追加渲染；**中断生成**（取消订阅、置 `done`）；网络失败重试；完成态；重新生成。
- 消息 `status` 走 `sending→streaming→done/error`。

### 2.2 登录完善 + 云同步抽象

- 邮箱登录流程打磨（错误提示、loading、退出登录确认）。
- `SyncService` 接口：`push/pull Favorites/History`；无后端时本地 no-op，但**调用时机与冲突策略**（以最新 `updatedAt` 为准）先定义好，便于接后端。
- 登录后触发一次 pull/merge（本地实现为合并去重占位）。

### 2.3 导师对比（`features/compare`，开放问题 Q6=是）

- 入口：详情页「加入对比」/ 结果页多选 2 位 → `/compare`。
- 并排对比：学校/学院/职称、研究方向、匹配点、推荐理由、limitations；差异高亮。
- 数据来自已加载的 `Recommendation`/`Professor`（必要时 `getProfessor` 补全）。

### 2.4 反馈闭环

- 提交后状态可见；可选「我的反馈」列表（本地）。

### 2.5 埋点与崩溃监控

- `AnalyticsService` 在关键路径打点（§7 事件全覆盖）；默认 Stub，预留 Firebase/友盟 接入。
- 崩溃监控接入点（Sentry/Crashlytics）预留初始化位与开关；比赛阶段可不真正上报。

### 2.6 隐私合规与设置页（`features/settings`）

- 设置页：登录/退出、清除缓存、清除历史、关于我们、隐私政策、用户协议、版本号、问题反馈。
- 隐私政策/用户协议/关于为静态页；提供**数据删除入口**（清空本地收藏/历史/缓存/token）。
- 提示用户勿输入敏感个人信息（需求 §12.5）。

### 2.7 接真实后端准备

- 补全 `data/remote/Http*Repository`（用 `DioClient`，按需求 §9 契约实现请求/解析/错误映射）。
- `AppConfig.dataSource=http` 时全链路可走通（对接预留的 mock server 或真实后端均可）；切换零改 presentation。
- HTTPS-only、token 注入拦截器、错误码→`AppException` 在拦截器统一处理。

## 3. 验收标准

- 流式回答可逐字展示、可中断、可重试、有完成态。
- 登录/退出正常；`SyncService` 调用时机与合并策略已实现（本地占位可演示）。
- 可选 2 位导师并排对比，差异清晰。
- 关键事件均有埋点（Stub 可见）；崩溃监控接入点存在。
- 设置页各项可用；可清除本地全部数据并验证不再展示。
- 将 `dataSource` 切到 `http` 后，针对契约的请求/解析/错误映射在集成测试或 mock server 下走通。

---

## 4. 测试（TDD）

- **单元**：流式切片/中断逻辑；`SyncService` 合并去重与 `updatedAt` 冲突策略；对比页差异计算；`Http*Repository` 用 Dio mock 适配器验证请求构造、JSON 解析、错误码映射；数据删除清空。
- **Widget**：流式渲染与中断按钮；对比页并排展示；设置页清除操作确认流程；隐私/协议静态页可达。
- **集成**：`dataSource=mock` 与 `=http`（mock server）下核心链路一致性冒烟。
