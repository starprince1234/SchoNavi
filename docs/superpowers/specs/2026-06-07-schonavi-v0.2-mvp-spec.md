# SchoNavi V0.2 MVP 版本 Spec

- 版本：v1（2026-06-07）
- 依赖：`2026-06-07-schonavi-master-design.md` 与 `2026-06-07-schonavi-v0.1-prototype-spec.md`
- 目标：**形成可用闭环**——在 V0.1 推荐链路上补齐 登录/对话/收藏/历史/反馈/全套异常态，产出 Android APK 测试包。

---

## 1. 范围

### 本阶段做（In Scope）

1. 启动页 + 新手引导页（首启判断）。
2. 模拟登录 + 游客模式。
3. 对话追问页（非流式）。
4. Prompt 更新 / 重推 + 变化提示。
5. 教师主页跳转（系统浏览器）。
6. 本地收藏（列表/取消/持久化）。
7. 本地历史（自动生成/恢复/删除/清空）。
8. 用户反馈（多入口）。
9. 全套状态：空结果 / 网络异常 / 数据缺失 / 接口错误码映射。
10. Android APK 测试包。

### 本阶段不做（移交 V1.0）

流式对话、云端同步、导师对比、埋点正式接入、崩溃监控、隐私/协议/关于页、设置页完善。

## 2. 功能细化

### 2.1 启动页 + 引导页（`features/onboarding`）

- Splash：Logo/名称 + 加载动画；读 `seenOnboarding` 决定去 `/onboarding` 或 `/home`。
- 引导页：4 屏（需求 §7.2 文案）；下一步 / 跳过 / 开始使用；完成后写 `seenOnboarding=true`。

### 2.2 模拟登录 + 游客模式（`features/auth`）

- `MockAuthRepository`：`loginWithEmail(email, code)` 校验邮箱格式即返回假 `User`（token 存 LocalStore）；`continueAsGuest()` 返回 `isGuest=true` 用户。
- 登录页：邮箱输入 + 「获取验证码」（mock，本地任意码通过）+ 「登录」+ 「以游客身份继续」。
- **核心功能免登录可用**；仅「云同步」类操作（V1.0）才引导登录。登录态由 `authStatusProvider` 暴露，设置页可登录/退出。

### 2.3 对话追问页（`features/chat`，非流式）

- 入口：结果页/详情页「继续追问」，携带 `sessionId`（+ 可选 `professorId`）。
- `MockChatRepository.sendMessage`：依 `sessionId` 上下文与意图（为什么推荐/相似导师/只看某地/换方向/适合硕士吗）返回合理假回答；可返回澄清问题或 `relatedRecommendations`（嵌入卡片）。
- 页面：消息列表、输入框、发送、快捷问题按钮、加载态、重新生成（重发上一条）。
- Markdown 渲染回答（依主设计文档 §2 选定的 markdown 包）。

### 2.4 Prompt 更新 / 重推（`features/recommendation`）

- 结果页「修改条件」→ 编辑 prompt → 带 `sessionId` 重新 `getRecommendations`。
- 结果更新后提示**变化**：新增的条件、被替换/新增的导师（Mock 在 result 里给出 diff 提示文案即可）。查询理解与理由同步更新。

### 2.5 教师主页跳转

- 详情页/卡片「访问教师主页」→ `url_launcher` 系统浏览器打开 `homepageUrl`。
- 空链接 → 提示「暂无主页信息」；打开失败/非法 → 提示「主页可能已失效，可通过学校官网确认」并提供反馈入口（需求 §6.4）。

### 2.6 本地收藏（`features/favorite`）

- `MockFavoriteRepository`（LocalStore 持久化，JSON）：add/remove/isFavorite/list/watch。
- 卡片与详情页收藏按钮（状态切换即时反映，`watch()` 驱动）。
- 收藏页：列表、搜索、取消收藏、查看详情、访问主页；显示收藏时间/学校/学院/方向。
- 持久化：重启后仍在；取消后列表同步更新。

### 2.7 本地历史（`features/history`）

- `MockHistoryRepository`（LocalStore）：推荐完成后自动写一条（prompt/时间/sessionId/摘要/方向/导师数）。
- 历史页：列表（摘要/时间/方向/数量/状态）、点击**恢复会话/重推**、删除单条、清空全部。

### 2.8 用户反馈（`features/feedback`）

- 入口：推荐卡片、详情页、对话回答下方、设置页问题反馈。
- 以底部 sheet 呈现：反馈类型（§7.9 枚举）+ 文本；字段 `professorId?/sessionId?/feedbackType/feedbackContent/createdAt`。
- `MockFeedbackRepository.submitFeedback` → 成功 toast；失败提示重试。

### 2.9 全套状态（需求 §11）

- 空结果：放宽条件文案 + 快捷修改按钮。
- 网络/超时/500：`ErrorView` + 对应中文文案（§6.1 映射表）+ 重试。
- 数据缺失：「暂无信息」，不渲染 null。
- 各错误码（400/401/403/404/429/500）按映射表展示。Mock 提供场景开关用于演示与测试。

### 2.10 Android APK

- 确认 `minSdk=31`；`flutter build apk --debug`（或 profile）产出可安装测试包；真机 + MuMu/AS 模拟器冒烟。

## 3. 关键数据流（新增）

```text
对话：ChatPage ─send(sessionId,msg)─▶ ChatNotifier
   └ chatRepository.sendMessage → Result<ChatResult>{answer, relatedRecommendations}
   ◀ 追加 assistant 消息（含嵌入卡片）；失败→error 态 + 重新生成

收藏：任意页点收藏 ─▶ favoriteRepository.add/remove ─▶ LocalStore(JSON)
   FavoritePage ─watch()─▶ 实时列表

历史：推荐成功 ─▶ historyRepository.add(SearchHistory)
   HistoryPage 点条目 ─▶ 带 sessionId/prompt 重推或恢复会话
```

---

## 4. 验收标准（需求 §17 全量）

- 用户可登录或以游客继续；游客可用全部核心功能。
- 可继续追问并得到回答；若返回新推荐/澄清问题，UI 能展示/引导。
- 修改条件后推荐结果、查询理解、理由同步更新并提示变化。
- 正常 URL 可打开；空 URL 不报错；非法 URL 有提示。
- 收藏切页保持、重启仍在、取消即时同步；历史自动生成、可恢复、可清空。
- 反馈可提交，失败可重试，成功有确认。
- 断网/超时/500/空数据/异常 JSON 均有提示，App 不崩溃。
- 产出可安装 APK，并在真机与模拟器冒烟通过。

## 5. 测试（TDD）

- **单元**：`MockAuthRepository` 登录/游客；`MockChatRepository` 各意图返回；`FavoriteRepository` add/remove/isFavorite/持久化；`HistoryRepository` 增删清；错误码→`AppException`→文案映射；Prompt 重推 diff 逻辑。
- **Widget**：引导页流程；登录校验 + 游客入口；对话发送/加载/重新生成；收藏切换与收藏页同步；历史恢复；反馈 sheet 提交成功/失败；空结果与各错误态文案。
- **集成（可选）**：输入→推荐→详情→追问→收藏→历史 全闭环冒烟。

全闭环可用、上述测试全绿、`flutter analyze` 无错、APK 可安装并在真机/模拟器通过 §18 主要场景。
