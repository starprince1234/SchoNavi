# SchoNavi V0.1 原型版本 Spec

- 版本：v1（2026-06-07）
- 依赖：`2026-06-07-schonavi-master-design.md`（架构/模型/契约/错误体系/目录均以主设计文档为准，本文不重复）
- 目标：**跑通核心推荐链路**——验证「自然语言输入 → 推荐结果 → 导师详情」端到端可用（数据走 Mock）。

---

## 1. 范围

### 本阶段做（In Scope）

1. **工程地基**：`ProviderScope`、亮/暗主题、`go_router` 骨架、`AppConfig`、`Result<T>` + `AppException` 体系、`core/di` 仓储绑定（默认 mock）、`DioClient` 骨架（预留不调用）、`LocalStore` 骨架。
2. **领域模型与 DTO**：`Professor`、`Recommendation`、`QueryUnderstanding`、`RecommendationResult` 及其 DTO（fromJson/toJson）。
3. **Mock 数据与仓储**：`MockDb` + fixtures（≥12 位中文导师）；`MockRecommendationRepository`、`MockProfessorRepository`。
4. **首页 / 推荐输入页**：输入框、示例 prompt、快捷标签、输入校验、开始推荐、清空。
5. **推荐结果页**：查询理解模块、推荐卡片列表、匹配等级 chip、推荐理由摘要、点击进入详情、加载/空/错三态。
6. **导师详情页**：基础信息、研究方向、bio、推荐理由、匹配点、limitations/数据质量提示、缺失字段「暂无信息」。
7. **共享三态组件**：`LoadingView`、`EmptyView`、`ErrorView`。

### 本阶段不做（Out of Scope，移交 V0.2/V1.0）

登录/游客、对话追问、Prompt 重推提示、收藏、历史、反馈、教师主页跳转、流式、设置、引导页、对比。
（注：详情页此阶段**隐藏**「访问主页/收藏/继续追问」按钮——这些交互在 V0.2 引入，本阶段不做半成品。）

## 2. 功能细化

### 2.1 工程地基

- `minSdk` 在 `android/app/build.gradle.kts` 显式改为 **31**。
- `lib/main.dart` 改为 `runApp(const ProviderScope(child: SchoNaviApp()))`，移除默认计数器 demo。
- `app.dart`：`MaterialApp.router` + 亮/暗主题（靛蓝/青绿 `ColorScheme.fromSeed`）+ 路由。
- `AppConfig`：`dataSource = mock`、`appVersion`、`featureFlags{ showMatchScore:false }`。
- `core/di/providers.dart`：`recommendationRepositoryProvider`、`professorRepositoryProvider` 按 `dataSource` 切换（当前恒为 mock）。

### 2.2 首页 / 推荐输入页（`features/home`）

- 顶部标题 + 标语（「用自然语言找到适合你的导师」）。
- 多行输入框；示例 prompt（需求 §7.3 五条）点按填入；快捷标签（方向/地域/阶段 chip）点按追加。
- **输入校验**：空 → 提示「请输入你的需求」并禁用按钮；过短（<6 字）→ 软提示「可补充方向或地区」；过长（>1000 字）→ 限制并提示。
- 「开始推荐」→ 调 `recommendationProvider.submit(prompt)` → 跳转结果页；「清空」复位输入。
- 「最近搜索」此阶段**留空占位区、不接数据**（V0.2 接历史）。

### 2.3 推荐结果页（`features/recommendation`）

- 顶部「查询理解」模块：研究方向 / 地域 / 学历阶段 / 其它偏好；`uncertainties` 以提示样式列出（需求 §7.4）。
- 推荐卡片列表（`shared/widgets/ProfessorCard`）：姓名、学校、学院、职称、研究方向 chip、`MatchLevelChip`、推荐理由摘要、（主页/收藏入口此阶段隐藏或置灰）。
- `matchScore` 不展示（featureFlag 关）。
- 四态：加载（`LoadingView`）、空（`EmptyView`，文案见需求 §11.2 + 返回修改）、错误（`ErrorView` + 重试）、数据（列表）。
- 点卡片 → `/professor/:id`。

### 2.4 导师详情页（`features/professor`）

- 经 `professorRepositoryProvider.getProfessor(id)` 拉取。
- 基础信息：姓名/学校/学院/职称/研究领域/bio/数据来源/更新时间/（主页链接展示但此阶段不跳转）。
- 推荐解释：匹配点、limitations / 数据质量提示。
- 缺失字段统一显示「暂无信息」，绝不空白或 null。
- 加载/错误态复用共享组件。

## 3. 数据流

```text
HomePage ──submit(prompt)──▶ RecommendationNotifier
   └─ ref.read(recommendationRepositoryProvider).getRecommendations(prompt)
        └─ MockRecommendationRepository → MockDb.match(prompt)
             → Result.success(RecommendationResult{ sessionId, queryUnderstanding, recommendations[], followUpQuestions[] })
   ◀── state: loading → data/empty/error
RecommendationPage 渲染 → 点击卡片 → go('/professor/:id')
ProfessorPage ── getProfessor(id) ── MockProfessorRepository → Result<Professor> → 渲染
```

---

## 4. 验收标准（对应需求 §17）

- 输入自然语言并点「开始推荐」，能在 3–10s 内得到推荐结果（Mock 含合理延迟）。
- 结果含：导师姓名/学校/学院/方向/匹配等级/推荐理由；理由非空。
- 查询理解模块正确展示解析到的方向/地域/阶段。
- 点击卡片进入详情，可见完整信息；缺失字段显示「暂无信息」。
- 空输入不可提交；空结果有明确提示与返回修改入口；Mock 注入错误时有错误态 + 重试，App 不崩溃。

---

## 5. 测试（TDD，先测后写）

- **单元**：四个 DTO 的 fromJson↔toJson 往返；`MockDb.match` 对不同 prompt（含方向+地域）返回非空且排序合理；`RecommendationNotifier` 状态流转 loading→data / loading→empty / loading→error（用 `ProviderContainer` 覆写注入假仓储）。
- **Widget**：空输入禁用按钮；结果列表渲染 N 张卡片；`EmptyView`/`ErrorView` 文案与重试回调；详情页缺失字段显示「暂无信息」。

---

## 6. 退出条件（进入 V0.2 的前提）

地基 + 三页跑通、`flutter analyze` 无错、上述测试全绿、可在 Android 模拟器演示「输入 → 推荐 → 详情」。
