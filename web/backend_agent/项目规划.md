## 一、项目架构

### 1.1 总体架构

采用**模块化单体应用（Modular Monolith）**架构，按领域模块解耦，为未来拆分微服务预留边界。

```

                      ↓
┌────────────────────────────────────────────┐
```

### 1.2 核心设计思想

```text
用户输入
  ↓
意图解析（LLM/规则）
  ↓
SQLite 结构化过滤（学院/职称/研究方向）
  ↓
ChromaDB 语义召回（导师简介语义匹配）
  ↓
NetworkX 图谱扩展（同院/同领域/合作关系）
  ↓
Python 规则重排（加权打分）
  ↓
LLM 生成推荐理由（可降级）
  ↓
返回推荐导师列表 + 匹配分数 + 推荐理由 + 关系路径
```

### 1.3 关键抽象

系统核心抽象为 **User 推荐 Item**：

```text
当前：Item = 导师 (Professor/Mentor)
未来：Item = 竞赛 / 课程 / 项目 / 论文 / 实习岗位
```

## 四、数据库设计（适配 scu.edu.cn.db）

### 4.1 原始数据源（scu.edu.cn.db）

| 表名 | 说明 | 关键字段 |
|---|---|---|
| `professors` | 教师信息 | name, org_unit_name, title, research_areas, email, bio, publications |
| `org_units` | 院系/机构 | name, url, kind |
| `academicians` | 院士信息 | name, title, research_areas, org_unit_id |
| `professor_affiliations` | 教授-院系关联 | professor_id, org_unit_id |
| `university_meta` | 大学元信息 | name, start_url, location |

### 4.2 业务数据库（app.db）

在原始数据基础上扩展：

| 表名 | 说明 | 关键字段 |
|---|---|---|
| `users` | 用户 | id, username, research_interests, created_at |
| `items` | 推荐物品（导师抽象） | id, external_id, title, category, tags, vector_id |
| `tags` | 标签 | id, name, type |
| `item_tags` | 物品-标签关联 | item_id, tag_id, weight |
| `interactions` | 用户行为 | user_id, item_id, action_type, weight |
| `graph_edges` | 图谱边 | source_id, target_id, relation, weight |
| `recommendation_logs` | 推荐日志 | request_id, user_id, strategy, latency_ms |
| `feedbacks` | 用户反馈 | user_id, item_id, feedback_type |
| `user_profiles` | 用户画像 | user_id, preferred_tags, summary |

### 4.3 知识图谱模型

**实体类型：**

| 类型 | 示例 |
|---|---|
| User | User_1001 |
| Professor | Prof_23 |
| OrgUnit | Org_计算机学院 |
| Tag | Tag_NLP |
| ResearchArea | RA_机器学习 |
| Paper | Paper_123 |

**关系类型：**

| 关系 | 含义 |
|---|---|
| works_at | 教授-所属院系 |
| has_tag | 教授-标签 |
| researches | 教授-研究方向 |
| similar_to | 教授-相似教授 |
| co_department | 同院系 |
| co_area | 同研究方向 |
| liked | 用户-喜欢教授 |
| viewed | 用户-浏览教授 |
