
根据当前前端代码推导：项目现在还是 `mock` 数据源，HTTP 仓储未接入；下面是前端页面实际需要的后端接口契约。收藏、历史记录目前是本地 `SharedPreferences`，不是后端接口。

**接口文档**

### 1. 获取导师推荐

接口名：`getRecommendations`

建议接口：`POST /api/recommendations`

请求体：

```json
{
  "prompt": "我想找医学影像和计算机视觉方向的导师，最好在上海，适合申请硕士。",
  "session_id": "s_xxx"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
|---|---:|---:|---|
| `prompt` | string | 是 | 用户自然语言需求 |
| `session_id` | string | 否 | 多轮会话 ID，首次推荐可不传 |

响应体：

```json
{
  "session_id": "s_123456",
  "query_understanding": {
    "research_interests": ["医学影像", "计算机视觉"],
    "preferred_locations": ["上海"],
    "preferred_universities": [],
    "degree_stage": "硕士",
    "uncertainties": ["未明确偏理论还是应用"]
  },
  "recommendations": [
    {
      "professor_id": "p_001",
      "name": "张三",
      "university": "上海交通大学",
      "college": "电子信息与电气工程学院",
      "title": "教授",
      "research_fields": ["医学影像", "计算机视觉", "深度学习"],
      "homepage_url": "https://example.edu.cn/zhangsan",
      "match_level": "高",
      "match_score": 0.92,
      "reason": "公开资料显示其研究方向与用户需求高度相关。",
      "limitations": ["公开资料中未明确招生信息"]
    }
  ],
  "follow_up_questions": [
    "你更倾向理论研究还是应用研究？",
    "是否只考虑 985 / 双一流高校？"
  ]
}
```

### 2. 获取导师详情

接口名：`getProfessor`

建议接口：`GET /api/professors/{professor_id}`

请求体：无

路径参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---:|---:|---|
| `professor_id` | string | 是 | 导师 ID，例如 `p_001` |

响应体：

```json
{
  "professor_id": "p_001",
  "name": "张三",
  "university": "上海交通大学",
  "college": "电子信息与电气工程学院",
  "title": "教授",
  "research_fields": ["医学影像", "计算机视觉", "深度学习"],
  "bio": "主要研究医学影像分析与深度学习在临床中的应用。",
  "homepage_url": "https://example.edu.cn/zhangsan",
  "source_url": "https://example.edu.cn/zhangsan/source",
  "updated_at": "2026-06-01",
  "data_quality_score": 0.87
}
```

### 3. 发送追问消息

接口名：`sendMessage`

建议接口：`POST /api/chat/messages`

请求体：

```json
{
  "session_id": "s_123456",
  "message": "为什么推荐这位导师？",
  "professor_id": "p_001"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
|---|---:|---:|---|
| `session_id` | string | 是 | 推荐结果或导师详情页进入聊天时携带的会话 ID |
| `message` | string | 是 | 用户追问内容 |
| `professor_id` | string | 否 | 如果从导师详情页进入聊天，则传当前导师 ID |

响应体：

```json
{
  "session_id": "s_123456",
  "answer": "推荐该导师的主要依据是研究方向匹配度较高，且公开资料显示其方向与用户需求相关。",
  "related_recommendations": [
    {
      "professor_id": "p_002",
      "name": "李娜",
      "university": "清华大学",
      "college": "计算机科学与技术系",
      "title": "副教授",
      "research_fields": ["自然语言处理", "大模型", "知识图谱"],
      "homepage_url": "https://example.edu.cn/lina",
      "match_level": "中",
      "match_score": 0.76,
      "reason": "研究方向与当前追问主题相近。",
      "limitations": ["主页链接可能变更，请以学校官网为准"]
    }
  ]
}
```

**补充约定**

前端 DTO 当前期望后端直接返回上述 JSON 对象，不是 `{ code, message, data }` 包裹格式。  
`match_level` 建议固定为：`高`、`中`、`低`。  
可选字段允许为 `null` 或省略：`homepage_url`、`match_score`、`bio`、`source_url`、`updated_at`、`data_quality_score`、`degree_stage`、`professor_id`。

另外我已按项目 AGENTS 约束补充了 `.gitignore` 里的 Doppler/Env 安全规则，没有写入任何环境变量或密钥。
